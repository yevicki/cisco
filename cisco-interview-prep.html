Practice Questions
🤖 AI Coding
Start Here · Most Nervous
"Walk me through how you actually use AI tools in your day-to-day development work." +
Be specific and honest. Mention concrete tools (GitHub Copilot, ChatGPT, Cursor, Tabnine) and concrete use cases: generating boilerplate, writing unit tests, explaining unfamiliar APIs, debugging stack traces, refactoring code. Then add nuance — talk about when you don't trust it (security-sensitive code, complex business logic) and how you verify its output. This shows maturity, not just enthusiasm.
💡 Tip: If you mostly use ChatGPT for code help, that's fine — just describe it precisely. "I paste in a failing test and ask it to explain why" is a great answer.
"Have you ever integrated an AI API into an application? Tell me about that experience." +
If yes: describe the API (OpenAI, Anthropic, Azure AI), the use case, how you handled API keys securely, rate limits, error handling, and cost. If no: be honest but pivot — "I haven't integrated one in production yet, but I've experimented with [X] and understand the fundamentals: authentication, prompt design, handling streaming responses, and cost management." Then show you know what it takes.
💡 Tip: Even a side project using the OpenAI API counts. If you have time before the interview, build a tiny demo.
"What do you know about Retrieval-Augmented Generation (RAG) and how might it apply to an HR platform?" +
RAG = giving an LLM access to your own documents at query time instead of retraining it. For HR: an employee asks "what's our parental leave policy?" — RAG retrieves the relevant policy document from a vector database and feeds it to the LLM as context, so the answer is accurate and up-to-date. Key components to mention: embedding model, vector store (Pinecone, pgvector, Weaviate), retrieval step, and generation step. You don't need to have built this — understanding the concept is enough at this stage.
💡 Tip: Cisco's HR AI platform almost certainly uses something like this for policy Q&A. Mentioning that connection impresses interviewers.
"How do you think about bias and fairness when AI is used in HR workflows?" +
This is critical for HR AI and shows you understand the domain. Key points: AI models trained on historical HR data can encode past biases (hiring, performance reviews, promotions). Mitigation approaches: audit datasets for representation gaps, use fairness metrics (demographic parity, equalized odds), involve diverse reviewers in the loop, keep humans in decision-making for consequential actions. Also mention regulatory context: EEOC guidelines apply, some states regulate AI in hiring. Show you take this seriously.
💡 Tip: This question tests domain awareness, not engineering skill. A thoughtful answer here can differentiate you from purely technical candidates.
"What's prompt engineering and when have you used it?" +
Prompt engineering is designing the input text to an LLM to get reliable, high-quality outputs. Techniques to mention: system prompts (setting context/persona), few-shot examples (show the model what you want), chain-of-thought (ask it to reason step by step), output formatting (ask for JSON, bullet points, etc.), and temperature control (lower = more deterministic). Give a concrete example even if small: "I use system prompts when querying ChatGPT for code to constrain it to a specific framework."
"How do you handle hallucinations or incorrect AI output in production?" +
Never trust AI output blindly in production. Strategies: grounding — use RAG to anchor answers in source documents; confidence thresholds — if the model is uncertain, fall back to human review; output validation — parse and validate structured outputs before using them; human-in-the-loop — for sensitive HR decisions, always require a human sign-off; monitoring — log inputs and outputs, track user corrections over time.
💡 Tip: For HR specifically, emphasize human-in-the-loop. No one wants an AI making autonomous hiring or termination decisions.
☕ Java
Technical Panel
"What's the difference between an interface and an abstract class? When would you use each?" +
Interface: defines a contract, no state (before Java 8), supports multiple inheritance, use when unrelated classes share behavior. Abstract class: can have state and concrete methods, single inheritance, use when classes share code and an "is-a" relationship makes sense. Post Java-8: interfaces can have default and static methods, which narrows the gap — but abstract classes still own state.
💡 Example: Runnable is an interface (anything can run). Vehicle might be abstract (Car and Truck share engine logic).
"Explain the Java Streams API. Write a stream that filters a list of employees and returns their names sorted." +
Streams process collections in a declarative, functional style. They're lazy (operations run only at terminal). Example:

employees.stream()
  .filter(e -> e.isActive())
  .map(Employee::getName)
  .sorted()
  .collect(Collectors.toList());

Key concepts: intermediate ops (filter, map, sorted) vs terminal ops (collect, forEach, reduce). Parallel streams exist but have overhead.
"How does HashMap work internally in Java?" +
HashMap uses an array of buckets. Key is hashed with hashCode() then hash(), then index = hash % capacity. Collisions are handled by chaining (linked list in each bucket, tree after 8 entries in Java 8+). Default capacity is 16, load factor 0.75 — when 75% full, it doubles and rehashes. Thread safety: not thread-safe; use ConcurrentHashMap for concurrent access.
🌱 Spring Boot
Technical Panel
"Explain @Transactional. What happens if a checked exception is thrown inside a @Transactional method?" +
@Transactional wraps a method in a database transaction. By default, Spring only rolls back on unchecked exceptions (RuntimeException and Error). A checked exception will NOT trigger rollback unless you specify rollbackFor. Common gotcha: calling a @Transactional method from within the same class bypasses the proxy — the transaction won't apply.
💡 Tip: This is a very common interview trap. Know the checked vs unchecked rollback difference cold.
"How would you design a secure REST API in Spring Boot for an HR platform?" +
Cover: Authentication (JWT via Spring Security, or OAuth2/OIDC with Cisco's SSO), Authorization (@PreAuthorize with roles — HR managers see more than employees), Input validation (@Valid, @NotNull, custom validators), HTTPS only, Rate limiting, Audit logging (who accessed what employee data and when — critical for compliance), CORS configured properly. Mention GDPR/data sensitivity since it's HR.
🗄️ Databases
Technical Panel
"What is the N+1 query problem and how do you solve it in JPA?" +
N+1: you query for N records, then for each one, JPA fires an additional query to load a related entity = N+1 total queries. Example: load 100 employees, then load each employee's department separately. Solutions: JOIN FETCH in JPQL, @EntityGraph, or batch fetching with @BatchSize. Use eager loading carefully — it can cause its own problems with large associations.
"When would you use a NoSQL database over a relational one?" +
NoSQL when: schema is flexible or evolving rapidly (document stores like MongoDB), extremely high write throughput (Cassandra), hierarchical/graph relationships (Neo4j), caching/sessions (Redis). Relational when: data integrity is critical, complex joins are needed, ACID guarantees matter. For HR: core employee records = relational (structured, compliance-sensitive). AI interaction logs or unstructured feedback = could be document store.
